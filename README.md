PackMan 1.0
Это лишь база, тестовый вариант так сказать
Здесь полностью отсутсвуют враги(призраки)
Здесь отсутсвует возможность победить(кроме как как собирать точки)
Здесь отсутствует оптимизация, потому что вместо массива char[,] list<string> и соответственно StringBuilder и сопутствующие расходы на оптимизацию, пусть код и стал читаеме
Отсутсвует многопоточность(отдельно карта, отдельно движение игрока(соотвественно врагов если бы они были))
Есть зависимости в методах(и довольно таки крупный метод для вычисления состояния игрорка при смене позиции)


PackMan 2.0
Наличие врагов идущих до игрока по принципу bfs(пока без ограничений радиуса)
Пока нельзя победить(хотя сделать это легко, но на практике выйграть слишком трудно))
Призраки выполняют свой поиск в отдельных потоках - поэтому количество призраков не влияет на оптимизацию
Большая часть кода более не менее приемлемо(наверное) расскидана по отдельным классам
В целом - гораздо лучше PackMan 1.0 )
